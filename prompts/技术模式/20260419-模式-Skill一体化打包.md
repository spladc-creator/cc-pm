## 模式名称

**Skill 一体化打包**

**定义**：Claude Code 的 skill 不只放文档，而是将 hook 脚本、配置模板等功能组件一起打包在 `.skill` 目录内

---

## 模式描述

### 问题

初始做法：skill 目录只放一个 `skill.md`（参考文档）。用户安装后还需要手动创建 hook 脚本、配置 settings.json，才能获得完整功能。这导致"安装了但没完全安装"。

### 解法

```
skill-name.skill/
  skill.md              # 技能说明 + Setup 章节给 Claude 的配置指令
  hooks/
    context-provider.py # hook 脚本，用动态路径定位依赖
```

**skill.md 的 Setup 章节**告诉 Claude 安装后应自动执行的步骤：

1. 检查 hook 脚本是否存在
2. 读取并合并 settings.json
3. 验证 hook 工作
4. 通知用户

**hook 脚本**用 `os.path.realpath(__file__)` 动态定位依赖，不硬编码绝对路径：

```python
SCRIPT_DIR = os.path.dirname(os.path.realpath(__file__))
TOOL_PATH = os.path.normpath(os.path.join(SCRIPT_DIR, '..', '..', '..', '..', 'bin', 'tool.sh'))
```

### 关键原则

1. **安装 = 功能完整**：用户不需要额外操作就能获得完整体验
2. **动态路径**：脚本通过 `realpath` 追溯 symlink 真实位置，任何项目通用
3. **Claude 可执行**：Setup 章节写给 Claude 的指令，Claude 读到后自动完成配置

---

## 反面经验

最初把 hook 脚本放在 `.claude/hooks/` 独立目录，skill.md 只放文档。用户指出"这不是功能实现"。教训：skill 的价值不只在于 Claude 知道什么，还在于它自带什么。
