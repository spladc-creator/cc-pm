## 模式名称

**三级路径自定位**

**定义**：CLI 工具通过"环境变量 > 标记文件 > 回退默认值"三级策略自动定位关键路径，无需用户手动配置

---

## 模式描述

### 问题

CLI 工具需要知道两个关键路径：工作目录（项目在哪）和资源目录（配置/数据在哪）。硬编码路径只能在一台机器上工作，每个用户手动配置则增加使用门槛。

### 解法

对每个关键路径，按优先级检测：

| 优先级 | 方式 | 适用场景 |
|--------|------|----------|
| 1 | 环境变量 | CI/CD、自动化脚本、高级用户 |
| 2 | 标记文件 | 在项目根目录放一个空文件，工具向上查找 |
| 3 | 回退默认 | 相对于脚本自身位置推导，或用 `~/.toolname/` |

```bash
detect_base() {
  # 1. 环境变量
  [[ -n "${TOOL_BASE:-}" ]] && [[ -d "$TOOL_BASE" ]] && { cd "$TOOL_BASE" && pwd; return; }
  # 2. 标记文件
  local dir="$(pwd)"
  while [[ "$dir" != "/" ]]; do
    [[ -f "$dir/.tool-base" ]] && { echo "$dir"; return; }
    dir="$(dirname "$dir")"
  done
  # 3. 回退
  cd "$SCRIPT_DIR/../.." && pwd
}
```

### 为什么有效

- **环境变量**：覆盖力最强，适合自动化场景
- **标记文件**：类似 `.git` 的约定，用户无感知，`cd` 到项目根就自动生效
- **回退默认**：零配置也能用，降低入门门槛

---

## 关联模式

- `git` 用 `.git` 目录标记仓库根
- `node` 用 `package.json` 标记模块根
- `direnv` 用 `.envrc` 标记环境配置
- Rust 的 `cargo` 向上查找 `Cargo.toml`

---

## 实践案例

cc-pm.sh 重构中：
- `CC_PM_BASE` > `.cc-pm-base` 标记文件 > 脚本上两级目录
- `CC_PM_REGISTRY` > 脚本旁 `prompts/` > `~/.cc-pm/registry/`

重构后 cc-pm.sh 从"只能在 spladc.com 内部使用"变为"可部署在任意机器上"。
